# Customer Segmentation (K-Means)

Проект по сегментации клиентов торгового центра с помощью алгоритма **K-Means** (unsupervised learning).

Используется классический датасет **Mall Customers**: для каждого клиента известны пол, возраст, годовой доход и оценка трат (Spending Score).

---

## Цель проекта

Разбить клиентов на группы (кластеры) по схожему поведению, чтобы:
- понять, какие типы клиентов посещают торговый центр;
- выделить VIP-клиентов и потенциальные точки роста;
- получить интерпретируемые сегменты для маркетинга.

---

## Данные

Файл: `data/Mall_Customers.csv`

Основные признаки:
- `Age` — возраст клиента;
- `Annual Income (k$)` — годовой доход в тысячах долларов;
- `Spending Score (1-100)` — оценка трат, выставленная торговым центром.

В кластеризацию идут два признака:
- годовой доход;
- оценка трат.


---

## Как запустить

1. Создать и активировать виртуальное окружение:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate  # Windows
   ```

2. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Запустить основной скрипт:
   ```bash
   python main.py
   ```

Скрипт:
- загрузит данные;
- выберет признаки и масштабирует их;
- подберёт оптимальное количество кластеров K методом локтя;
- обучит модель K-Means;
- построит и сохранит графики в папку `reports/`.

---

## Результаты

Метод локтя показал оптимальное число кластеров **K = 5**.

Краткая интерпретация сегментов (по среднему доходу и тратам):

- **Кластер 0** — клиенты со средним доходом и средними тратами (основная аудитория);
- **Кластер 1** — богатые клиенты с высокими тратами (VIP-сегмент);
- **Кластер 2** — клиенты с низким доходом, но высокими тратами (лояльные к магазину);
- **Кластер 3** — клиенты с высоким доходом и низкими тратами (потенциал для маркетинга);
- **Кластер 4** — клиенты с низким доходом и низкими тратами (наименее приоритетная группа).
